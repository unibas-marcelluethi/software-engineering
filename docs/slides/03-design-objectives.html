Was ist design

Formgebende und funktionale Gestaltgebung eines Produkts.
  • Strukturiert ein Artefakt
  • Zerlegen des Gesamtsystems in (einfachere) Komponenten
    • Zuweisen von Verantwortlichkeiten
    • Sicherstellen, dass das Gesamtsystem Anforderungen erfüllt
  • Nicht primär ästhetische Aufgabe

Beispiel: Architektur

  • Entwurf eines Einkaufszentrums
  • Teile:
    • Parkplatz
    • Heizungs/Lüftungssystem
    • Verkabelung

Beispiel Lehre

  • Entwickeln einer Lehrveranstaltung
  • Teile:
    • Einzelne Lektionen
    • Übungen
    • Prüfung

Software design

Software design: Aufteilung eines Systems in Module
  • Ouput: Design Dokument
    • Beschreibt einzelne Module
    • Definiert und beschreibt Schnittstellen

Module

Ein Modul ist ein wohldefinierter Teil einer Software
  • Softwarearchitektur: Definiert Struktur der Module
  • Moduldesign: Entwurf individueller Module

Module können selbst wieder aus Modulen bestehen

  • Keine strikte Trennung zwischen Architektur und Design möglich

Ziel eines Designs

Struktur so festzulegen, dass hohe Softwarequalität erreicht werden kann.
Besonders wichtig:
  • Änderungen ermöglichen
    • Wahrscheinliche Änderungen sollten einfach zu implementieren sein.

Gründe für Änderungen

  • Änderung in Algorithmus
  • Änderung in Datenstruktur
  • Neue Benutzeranforderung
  • Wechsel der virtuellen Maschine
    • Betriebssystem, Browser, DBMS
  • Änderungen im Umfeld
    • Beispiel: Einführung Euro
  • Änderungen aus Entwicklungsprozess
    • Erkenntnisse aus Rapid Prototyping

"There are two ways of constructing a software design: One way is to make it so simple that there are obviously no deficiencies, and the other way is to make it so complicated that there are no obvious deficiencies. The first method is far more difficult."

Tony Hoare - 1980
