Ziel des Projekts

In kleinen Teams Ganttproject weiterzuentwickeln.
  • Praktische Anwendung der erarbeiteten Konzepte
  • Durchlaufen der Projektphasen
    • Anforderungsanalyse
    • Design und implementation einer Erweiterung / Verbesserung in existierendes System
    • Testen und Integration
    • Dokumentationen erstellen
  • Im Team arbeiten.

Weitere Ziele

  • Ganttproject verbessern.
  • Spass haben.
  • Selbstbewusstsein erlangen:
    "Ja, ich kann auch an einem komplexen Projekt mitarbeiten."

Gruppengrösse

  • Mindestens 2 Personen pro Team
  • Grosse Teams möglich
    • Jeder muss wohldefinierte Aufgabe haben
    • Punkte werden zum Teil individuell, zum Teil als Gruppe verteilt

Projektablauf

Projektidee 1: Probabilistisches Planen

  • Idee: Zeitschätzungen sind immer unsicher
    • Unsicherheit wird modelliert
    • Projektplan wird probabilistisch
  • Sehr flexible Idee: Von 2-4 Personen

Projektidee 2: Effort driven Scheduling

  • Idee: Zeit um Task zu erfüllen sollte von vorhandenen Ressourcen abhängen
    • Fehlendes Feature in Ganttproject
    • Dringendes Bedürfnis der Nutzer
    • Github Issue 83
  • Ideale Gruppengrösse: 2 Personen

Projektidee 3: Build system / Packagestruktur verbessern

  • Idee: Ganttproject Build ist sehr komplex und nicht standard
    • Buildsystem, code layout nach Gradle standards
    • Package struktur vereinheitlichen
  • Ideale Gruppengrösse: 2 Personen

Projektidee 4: Unit Tests / Code Coverage Export und Import

  • Idee: Codequalität von Export erhöhen
    • Exportfunktionalität ist schlecht getestet
    • Unit Tests müssen geschrieben werden
    • Automatische Code Coverage tools verwenden und in Buildsystem integrieren.
  • Ideale Gruppengrösse: 2 - 3 Personen

Projektidee 5: Deine Idee

  • Eigene Ideen sehr willkommen
    • Nur beschränkte technische Unterstützung durch Dozenten/Tutoren
  • Ideale Gruppengrösse: abhängig vom Projekt.

Eure Aufgabe

  • Gruppen bilden
  • Projektideen ausdenken
  • Projektidee auswählen / beschreiben
